# Home — план главной страницы

Главная отвечает на вопрос: **«Что происходит рядом со мной прямо сейчас и что делать?»**

Не дублирует Map (полная карта) и People (каталог с поиском). Это **дайджест + быстрые действия**.

---

## Цели экрана

| Цель | Как достигаем |
|------|----------------|
| Быстро понять активность рядом | Сводка цифрами + блоки контента |
| Включить своё presence | Первый кружок «Ты» в stories-ряду |
| Найти живых людей сейчас | Stories-кружки «Сейчас в кафе» сверху |
| Перейти к карте или людям | Ссылки в заголовках секций |
| Написать в Telegram | Кнопка на карточке человека |

---

## Структура сверху вниз

### 1. Stories — «Сейчас в кафе»

**Назначение:** live-люди в формате stories-кружков сверху экрана (как в Instagram / Telegram).

| Элемент | UI |
|---------|-----|
| Первый кружок «Ты» | Пунктир + `+` → открывает sheet статуса |
| Sheet | Toggle **«Онлайн и доступен»** + выбор кафе (необязательно) |
| Без кафе | Подпись «Доступен», live-обводка, не на карте кафе |
| С кафе | Подпись = название кафе, виден на Map |

**Визуал:**
- Градиентная обводка = человек сейчас в кафе и shared presence
- Горизонтальный скролл, edge-to-edge
- Подпись кафе мелким текстом под именем

**Данные:** `UserPresence` + users где `isVisibleOnMap() === true`.

---

### 2. Stats Row

**Назначение:** одним взглядом — масштаб активности рядом.

| Метрика | Источник | Тап |
|---------|----------|-----|
| Кафе рядом | `cafeService.nearby()` | → `/map` |
| Люди онлайн | `presenceService.countOnline()` | → `/people` |
| Сейчас в кафе | `presenceService.countInCafeShared()` | → `/map` |

Три компактные карточки в ряд.

---

### 3. Сейчас в кафе

Уже отображается в stories-ряду (блок 1). Отдельная секция с карточками не нужна.

---

### 4. Кафе рядом

**Назначение:** куда можно пойти; teaser для Map.

- Вертикальный список 3–4 кафе
- Карточка: название, расстояние, «N человек сейчас»
- Тап по карточке → `/map?cafe={id}` (позже)
- Заголовок + **«Все на карте»** → `/map`

**Данные:** `Cafe[]` + count presence per cafe.

---

### 5. Тебе могут подойти

**Назначение:** рекомендации из сообщества (не обязательно в кафе сейчас).

- 2–3 карточки людей
- Подбор по интересам / `matchService`
- Заголовок + **«Все люди»** → `/people`

**Отличие от блока 3:** там live «прямо сейчас», здесь — **подбор**.

---

### 6. Встречи (фаза 2)

**Назначение:** ближайшие meetups без отдельного таба.

- Список 1–2 событий: время, место, участники
- Кнопка «Создать встречу»
- Данные: `eventService.upcoming()`

Пока **не в MVP** — секция закомментирована или скрыта.

---

## Чего нет на Home

| Элемент | Почему | Где |
|---------|--------|-----|
| Полная карта | Отдельный экран | `/map` |
| Поиск / фильтры людей | Каталог | `/people` |
| Редактирование профиля | Редкое действие | `/settings` |
| Чат | Telegram Mini App модель | Telegram |

## Данные (MVP → prod)

| Секция | MVP (сейчас) | Prod |
|--------|--------------|------|
| Presence | mock / статичный UI | `presenceService.getMine()` |
| Stats | mock цифры | агрегация из repos |
| В кафе | mock users | `presenceService.listVisibleOnMap()` |
| Кафе | mock cafes | `cafeService.nearby(lat, lng)` |
| Рекомендации | mock users | `matchService.findMatches()` |

---

## Порядок реализации

1. **Сейчас** — UI-секции с mock-данными (заглушки)
2. Repositories: `cafe.repo`, `presence.repo`
3. Services + подключение к Home через Server Components
4. Кнопки presence («Я в кафе» / «Скрыть») — client actions
5. Геолокация + реальные кафе рядом
6. AI-рекомендации в блоке «Тебе могут подойти»
7. Блок встреч (фаза 2)
